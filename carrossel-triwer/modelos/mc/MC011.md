# MC011 — Caso de Aluno como Prova Central

**Definição**
Carrossel que usa o resultado de um aluno como veículo primário de autoridade. O aluno é o protagonista — o criador é a metodologia de fundo. A trajetória do cliente substitui a auto-declaração do criador como prova de sistema.

**Objetivo estratégico**
Autoridade por transferência. A audiência se identifica com o aluno (par) mais facilmente do que com o criador (autoridade distante). O resultado do aluno soa mais atingível do que o resultado do criador, reduzindo a barreira de crença ("isso funciona para outros, não só para quem criou"). Alta eficácia para quebrar objeção "só funciona para você".

## Sequência funcional slide a slide

| Slide | Função | Conteúdo típico |
|---|---|---|
| S01 | Resultado do aluno na headline | Número + contexto mínimo do perfil do aluno |
| S02 | Quem era esse aluno antes | Perfil inicial — sem glamour, sem subestimação exagerada |
| S03 | Diagnóstico do problema central | O que estava impedindo o resultado — específico, não genérico |
| S04–06 | O que foi aplicado | Método / intervenção nomeada — específica o suficiente para credibilidade |
| S07 | Prova tipo 1 | Dado de resultado — número, período, screenshot |
| S08 | Prova tipo 2 | Reação espontânea do aluno — depoimento literal, comportamento não solicitado |
| S09 | Generalização | O que este caso demonstra que é replicável para outros perfis |
| S10 | CTA | Como o leitor pode acessar o mesmo método |

---

## Critérios de reconhecimento
- O sujeito da headline é o aluno, não o criador
- A identidade do aluno é especificada o suficiente para credibilidade (perfil reconhecível, não personagem genérico)
- Prova dupla obrigatória: dado quantitativo + reação comportamental
- O método está presente mas não é o foco central — o resultado do aluno é

**DOPA principal:** Autoridade
**DOPA secundário:** Oportunidade (o caso demonstra que o resultado é acessível — não excepcional)

---

## Distinção crítica em relação a modelos limítrofes
- **vs MC007 (Processo Passo a Passo com Credencial):** MC007 começa pela credencial do criador e desenvolve o método do criador. MC011 começa pelo resultado do aluno. Em MC007 o criador é o protagonista; em MC011 o aluno é o protagonista e o criador é o método.
- **vs MC003 (Narrativa de Virada Pessoal):** MC003 é a história pessoal do criador com arco emocional. MC011 é a história do aluno — o criador aparece como facilitador e metodologia, não como personagem.
- **vs MC009 (Refutação com Prova Dupla):** MC009 parte de voz externa sobre o criador. MC011 parte do resultado de um terceiro (o aluno) como prova do método do criador.

## Quando usar
Posts de autoridade quando o criador tem resultados de alunos documentados com dados específicos. Funciona como prova de sistema — "não sou só eu que consigo". Alta eficácia pré-lançamento para reduzir objeção "funciona apenas para você / para quem já é grande". Melhor combinado com headline [[Triwer — MH015 Caso Clínico Numerado|MH015]] quando é parte de uma série.

## Assinatura de performance
Saves altos — guardado como referência de que o método funciona para outros. Comentários de "quero ser esse aluno" e "que perfil incrível". Alta taxa de DMs com "como consigo isso?". Conversão Manychat acima da média quando a CTA é um passo próximo ao que o aluno realizou.
