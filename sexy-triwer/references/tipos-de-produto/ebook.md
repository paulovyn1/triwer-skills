# Ebook

| | |
|---|---|
| **Exemplo real** | Imperium |
| **Entrega** | PDF |
| **Ticket comum** | R$97–297 |
| **Vantagem** | Ótimo produto perpétuo, vende muito facilmente |
| **Desvantagem** | Reembolso alto, muitas vezes não gera percepção de valor |
| **Posição no funil** | Produto de entrada e/ou orderbump |

Não deve ser feito como livro — precisa ser diagramado como post/carrossel,
pensado para ser consumido em celular ou tablet. A experiência de leitura
importa tanto quanto o conteúdo.

**Perguntas para estruturar:**
1. É fácil de ler?
2. O design incentiva a pessoa a ler?
3. O assunto é objetivo?

**Alerta:** se não for específico, dificilmente vende no tráfego pago.

**Pilares dominantes:** Avareza e Preguiça — ver `../pilares-do-desejo/`.
Formato de ticket baixo — a promessa precisa ser específica (uma situação
só), nunca ampla (ver regra estrutural em `INDEX.md`).
